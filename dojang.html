도장.
오프체인을 온체인으로.

GIWA Dojang은 신뢰할 수 있는 오프체인 정보를 검증된 온체인 어테스테이션(Attestation)으로 발급하는 GIWA Chain의 네이티브 신뢰 레이어입니다. SOVE는 모든 실물 자산의 흐름을 도장으로 봉인합니다.

01 — Concept

도장은 무엇인가.

도장(Dojang)은 승인된 발급자(Issuer)가 발급하는 온체인 어테스테이션입니다. 한국 전통의 인장이 신원과 결정을 봉인하듯, 디지털 도장은 오프체인 데이터의 진위를 온체인에 영구 기록합니다.

예를 들어 Identity Stamp는 KYC를 거친 지갑임을 증명하는 SBT 형태로 발급됩니다. 현재 GIWA 생태계에서 Identity Stamp는 Upbit Korea가 유일한 공식 발급자입니다. 같은 원리로 SOVE는 감정기관·수탁기관·보험사가 발급하는 추가 도장으로 실물 자산의 전 생애주기를 신뢰 체인 위에 올립니다.

03 — Stamp Types

SOVE에서 발급되는
6종 도장.

각 도장은 승인된 발급자만 생성할 수 있으며, 발급 시점·발급자 서명·대상 자산 ID가 온체인에 봉인됩니다. 모든 도장은 SBT(Soul Bound Token) 형태로 양도 불가합니다.

01
Appraisal Stamp
Issuer · Certified Appraiser
Records appraised value (USD), date, appraiser signature, and methodology permanently on-chain. Direct trigger for $SOVE Mint.
02
Custody Stamp
Issuer · Certified Custodian
Seals storage location, conditions (temp/humidity), intake time, and 1:1 physical-NFT matching. Renewed after monthly audits.
03
Insurance Stamp
Issuer · Insurer
Policy number, coverage (110% of appraisal), validity, insurer signature. Auto-renewal notice before expiry.
04
Identity Stamp
Issuer · Upbit Korea
GIWA Verified Address — KYC-passed SBT. One issuance grants automatic KYC across all dApps.
05
Redemption Stamp
Issuer · Custodian
Issued on physical redemption claim. Seals release confirmation, shipping info, and delivery completion — basis for permanent NFT burn.
06
Re-Appraisal Stamp
Issuer · Appraisal Committee
Annual scheduled re-appraisal. Adjusts market value + recalculates collateral ratio. Special re-appraisal upon holder request.
04 — Data Structure

도장 어테스테이션 구조.

DojangAttestation.sol
struct Attestation {
  bytes32 schemaUID;       // Dojang type identifier
  address issuer;          // Certified issuing institution
  address subject;         // Target asset or wallet
  uint256 assetId;         // SOVE asset ID
  uint256 issuedAt;        // Issuance timestamp
  uint256 expiresAt;       // Expiry (0 = perpetual)
  bytes32 dataHash;        // Off-chain data hash
  string  metadataURI;     // IPFS URI
  bytes   signature;       // Issuer signature
}

Verification Logic

01Schema ValidationCheck that schemaUID matches a registered Dojang type
02Issuer AuthorityConfirm the issuer has authority for that Dojang type
03Validity PeriodexpiresAt > block.timestamp
04Signature VerificationVerify signature was signed by issuer's private key via ecrecover
05Data HashConfirm SHA-256 of metadataURI = dataHash → valid
